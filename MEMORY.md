# MEMORY

这是记忆系统的顶层索引与说明。请参阅 AGENTS.md 以了解代理（Claw）的工作方式与规则。

目录结构（简短说明）：

- memory/long/: 长期记忆（设计决策、偏好、长期约定）。每个主题用独立文件保存，文件名使用小写短横分隔。
- memory/daily/: 每日/临时记录。按日期创建文件 memory/daily/YYYY-MM-DD.md，记录当天活动与短期要点。

使用说明：
- 所有长期或重要信息应写入 memory/long/ 下的文件，不要只保留在对话中。
- 每日记录应简短（1-6 行摘要）并包含元信息（created, tags）。
- 不在仓库中保存任何密钥、密码或敏感凭证；如需记录凭证位置，只写指向安全存储的引用（例如 Vault 路径）。

参见：AGENTS.md（仓库根）
